# 画面レイアウト案（v1）

[app-spec.md](app-spec.md) 9章「画面構成の詳細（ワイヤーフレーム）」に対応する検討資料。
策定経緯: [docs/plans/archive/screen-layout-ideas.md](../plans/archive/screen-layout-ideas.md)、
[docs/plans/archive/daily-flow-screen-redesign.md](../plans/archive/daily-flow-screen-redesign.md)
（毎日の使い方＝Lesson追加→写真アップロード→単語追加→問題を解く、を最短導線にする見直し）

[data-model.md](data-model.md) で確定したデータモデルを元に、必要な画面を洗い出し、
各画面のレイアウト案（テキストベースのワイヤーフレーム）をまとめる。
配色・フォント等のビジュアルデザインは対象外とし、情報設計・画面構成・遷移に絞る。

## 0. 全体ナビゲーション

```
[ Lessons ]  [ Words ]  [ Settings ]
```

（v2: 4タブ構成「ホーム/単語帳/問題/設定」から3タブに整理し、タブ表示名は英語とする。
経緯: [docs/plans/archive/tab-navigation-redesign.md](../plans/archive/tab-navigation-redesign.md)、
[docs/plans/archive/tab-labels-english.md](../plans/archive/tab-labels-english.md)）

- レッスンタブ: クラスとレッスンを選択すると、教材の画像・文字起こし・翻訳と、そのレッスンに
  関連付いた単語や問題が見られる（旧ホームタブ。「現在のレッスン」を中心にした日次フローの
  ダッシュボードで、クラス/レッスンのドリルダウンはせず切り替えシートで選択する）。
  問題（`Question` の一覧・出題・成績）は独立タブを持たず、このタブ内に統合する
- 単語タブ: `Word` の一覧・詳細・追加・復習（デフォルトは全体ビュー。追加時にレッスンを
  任意で指定でき、指定なしでも登録できる。仕様書4章「全体の一覧ビュー」に対応）
- 設定タブ: 母語（翻訳先言語）設定など（内容は今後拡充）

「現在のレッスン」は直近に作成・選択した `Lesson` を指す**端末内のUI状態**（例:
`@AppStorage` で `Lesson.id` を保持）であり、`data-model.md` へのフィールド追加は不要。
保持していた `Lesson` が削除済みの場合は最新の `Lesson` にフォールバックする。

## 1. 画面一覧・データ対応表

| # | 画面名 | 対応データ | 対応する仕様書セクション | 頻度 |
|---|---|---|---|---|
| 1 | ホーム（現在のレッスン） | `Lesson`（photos/wordOccurrences/questions件数） | 4章 | 毎日 |
| 2 | クラス/レッスン切り替え | `Class`[], `Lesson`[] | 4章（管理単位） | 低頻度 |
| 3 | 撮影 | `Photo`（新規作成） | 3.1章 | 毎日 |
| 4 | 写真一覧 | `Photo`[]（`Lesson`配下） | 3.1章 | 確認用 |
| 5 | 写真詳細（OCR・翻訳結果） | `Photo`（ocrText/translatedText） | 3.1章 | 毎日 |
| 6 | 単語帳一覧 | `Word`[]（全体 or `WordOccurrence`経由でレッスン絞り込み） | 3.2章, 4章 | 確認用 |
| 7 | 単語詳細 | `Word`（translation/exampleSentence/partOfSpeech/grammarNote/reviewState）, `WordOccurrence`[] | 3.2章 | 低頻度 |
| 8 | 単語登録（手動／OCRタップ） | `Word`（新規）, `WordOccurrence`（新規） | 3.2章 | 毎日 |
| 9 | 復習（フラッシュカード） | `Word`[]（`reviewState.dueDate`順） | 3.2章 | 任意 |
| 10 | 問題一覧 | `Question`[]（全体 or `Lesson`絞り込み） | 3.3章, 4章 | 確認用 |
| 11 | 出題（クイズ） | `Question`（type/prompt/choices） | 3.3章 | 毎日 |
| 12 | 採点結果・解説 | `QuizResult`（新規）, `Question.explanation` | 3.3章 | 毎日（出題に付随） |
| 13 | 成績・演習履歴 | `QuizResult`[]（集計、全体 or `Lesson`絞り込み） | 3.3章 | 低頻度 |
| 14 | 設定 | （アプリ設定。専用エンティティなし） | 6章 | 低頻度 |

- 「頻度」列は、フィードバックにあった毎日の4ステップ（Lesson追加／写真アップロード／単語追加／
  問題を解く）にどう対応するかの目安。「毎日」の画面（#1, #3, #5, #8, #11, #12）がホームから
  最短導線でつながることを2章で確認する
- `Photo.imageFileName` はファイルシステム参照のため、画面側では画像表示コンポーネントとしてのみ扱う
- 写真#5のOCR結果テキストは画面#8・#10への遷移元にもなる（タップで単語登録／問題生成のトリガー）
- 全エンティティ（Class / Lesson / Photo / Word / WordOccurrence / Question / QuizResult）が
  いずれかの画面から参照されていることを確認済み

## 2. 各画面のレイアウト案

### 2.1 レッスン（ヘッダーカード型）

（v2: 旧「ホーム」。ヘッダーカード型に作り直し。
経緯: [docs/plans/archive/lesson-screen-redesign.md](../plans/archive/lesson-screen-redesign.md)）

```
レッスン
┌─────────────────────────────┐
│ ESL Beginner A                    │ ← クラス名（小）
│ Unit 3 Reading                 ⇅ │ ← レッスン表示名（大）。タップで2.2 切り替えシート
│ Wed, Jul 8, 2026                  │ ← 授業日（タイトル付きレッスンのみ補足表示）
└─────────────────────────────┘
[📷 写真を追加]
── コンテンツ (3) ──────────────
[thumb] 10/12  完了
── 単語 (12)  ────────────────
apple   りんご                    >
── 問題      ────────────────
（今後実装）
```

- 表示要素: 選択中の `Class.name` / `Lesson.displayTitle`（タイトル未設定なら授業日）を
  リスト先頭のカードで常時表示。タイトル付きレッスンは授業日（`Lesson.date`）を補足表示。
  配下にコンテンツ（`photos`: 画像・文字起こし・翻訳）／単語（`wordOccurrences` 経由の
  重複排除済み一覧）／問題（`questions`。未実装のためプレースホルダ）のセクション
- 主要アクション: `[📷 写真を追加]` → 2.3 撮影 → 2.5 写真詳細、コンテンツ行タップ → 2.5、
  単語行タップ → Words タブへ切り替えて 2.7 単語詳細を表示（単語の閲覧・操作は単語タブに
  集約する。タブ切替は `AppRouter` が仲介）
- 切り替え・作成: ヘッダーカードのタップ → 2.2 切り替えシート（レッスンの選択・作成は
  シート内のカレンダーで行う。2.2参照）。カードの長押し（コンテキストメニュー）→
  「Rename Lesson」でレッスン名（任意ラベル）を編集
- 空状態: クラスなし → 「クラスを追加」ボタン（2.2を開く）、レッスンなし → カード＋
  「レッスンを追加」ボタン（2.2のカレンダーを開く）
- v1にあった `[📝 単語を追加]` `[✏️ 問題を解く]` のクイックアクションは未実装
  （単語追加は単語タブから行う。将来ここに追加する場合は本章を更新する）

### 2.2 クラス/レッスン切り替え（ボトムシート・カレンダー型）

（v3: レッスン一覧をクラスごとのカレンダーに置き換え。レッスンは「クラス内で1日1つ」で、
授業日（`Lesson.date`）が実質の識別子。経緯: [docs/plans/lesson-calendar.md](../plans/archive/lesson-calendar.md)）

```
┌─────────────────────┐
│ ESL Beginner A ⇕     ✎ │ ← クラス切り替えMenu（+ クラス追加）／✎ クラス名編集
│  July 2026        < > │
│  SUN MON ... SAT      │
│   5   6   7  (8)  9 …  │ ← レッスンのある日: ドット／現在レッスンの日: ✓
│ [ Create Today's Lesson ]│ ← 今日のレッスンを作成（既存なら開く）
│  ヒントテキスト           │
└─────────────────────┘

クラス作成フォーム:  クラス名 [            ] （追加）
レッスン作成確認:   アラート「New Lesson」＋ タイトル（任意）[      ] （Create / Cancel）
```

- 表示要素: 上部にクラス切り替え Menu（チェックマーク付き一覧＋「Add Class」）と
  クラス名編集ボタン、下部に選択中クラスの `UICalendarView`（`LessonCalendarView`）。
  レッスンのある日にドット、現在レッスンの日にチェックのデコレーション
- 主要アクション:
  - レッスンのある日をタップ → そのレッスンを選択してシートを閉じ、2.1 へ戻る
  - 空き日をタップ → 作成確認アラート（タイトルは任意入力。空なら表示名は授業日になる）→
    Create で作成・選択してシートを閉じる。**同日の重複作成は UI 上できない**
  - 「Create Today's Lesson / Open Today's Lesson」ボタン → 今日の日付で同上（作成 or 選択）
  - Menu の「Add Class」→ クラス作成フォーム（作成後はそのクラスのカレンダーに切り替わる）
- 遷移先: 選択・レッスン作成 → 2.1 レッスン
- レッスン名の変更は 2.1 ヘッダーカードのコンテキストメニューから（Edit Lesson フォーム。
  重複チェックなし・空可）。旧「レッスン作成フォーム」（`LessonAddView`）は廃止

### 2.3 撮影

```
┌─────────────────────┐
│                       │
│      カメラプレビュー   │
│                       │
└─────────────────────┘
        ( ● シャッター )
```

- 表示要素: カメラプレビューのみ（シンプルな撮影専用画面）
- 主要アクション: シャッターを押す → `Photo`（`processingStatus = pending`）を現在のレッスンに
  作成し、オンラインならOCR・翻訳リクエストを送信（`processing`へ遷移）
- 遷移先: 撮影後 → 2.5 写真詳細（処理中表示のまま遷移し、完了をその場で待てる）

### 2.4 写真一覧（レッスン内）

```
< Unit 3 Reading        [📷 追加撮影]
[thumb ✓][thumb ⏳][thumb ⚠]
```

- 表示要素: `Photo`サムネイル一覧、`processingStatus`バッジ（pending/processing/completed/failed）
- 主要アクション: サムネイルタップ、追加撮影、失敗写真の再試行
- 遷移先: タップ → 2.5 写真詳細

### 2.5 写真詳細（OCR・翻訳結果）

```
< 写真詳細
┌───────────┐
│  撮影画像    │
└───────────┘
─── OCR結果（英語）───
This is a [pen]. I have...
─── 翻訳（日本語）───
これは[ペン]です。私は...
```

- 表示要素: 撮影画像、`ocrText`、`translatedText`（並べて表示。仕様書3.1章要件）、
  `processingStatus`（処理中はローディング、失敗時は再試行ボタン）
- 主要アクション: OCRテキスト中の単語をタップ → 単語登録（`WordOccurrence.sourcePhoto`に
  この `Photo` を設定）、この写真から問題を生成
- 遷移先: 単語タップ → 2.8 単語登録（事前入力）、問題生成 → 2.10 問題一覧（生成後）

### 2.6 単語帳一覧

```
[全体 ▾] [レッスン: Unit 3 ▾]   [+ 手動登録]  [復習する →]
┌─────────────────────┐
│ apple   りんご         │→
│ book    本             │→
└─────────────────────┘
```

- 表示要素: `Word.text`/`translation` 一覧、検索フィールド（見出し語・訳語の部分一致、
  大文字小文字を区別しない。ヒット0件時は「結果なし」表示）、表示モード切り替え（全体／レッスン絞り込み。
  レッスン絞り込みは対象 `Lesson.wordOccurrences.map { $0.word }`）、`reviewState.dueDate`
  が今日以前の件数バッジ（復習対象数）
- 主要アクション: 単語タップ、手動登録、復習開始
- 遷移先: タップ → 2.7 単語詳細、手動登録 → 2.8、復習開始 → 2.9

### 2.7 単語詳細

```
< apple
りんご                         [🔊]
─────────────────────
例文: I ate an apple. (教科書より)
品詞: 名詞　文法: 可算名詞
─────────────────────
登場したレッスン
・Unit 3 Reading (10/12)
・Unit 1 Fruits (10/02)
─────────────────────
復習: 次回 10/15 ・ 3回目
```

- 表示要素: `text`/`translation`/`exampleSentence`（+`exampleSentenceSource`表記）/
  `partOfSpeech`/`grammarNote`/発音再生ボタン（TTS）、`occurrences`から
  `lesson.title`+`occurredAt`の一覧、`reviewState`（dueDate/reviewCount）
- 主要アクション: 発音再生、編集、削除（`Word`＋`occurrences`カスケード削除の確認）
- 遷移先: 登場レッスンタップ → 2.1 ホーム（該当レッスンを「現在のレッスン」にして表示）

### 2.8 単語登録（手動／OCRタップ）

```
< 単語を登録
見出し語: [apple            ]
訳語:     [りんご            ]
例文:     [I ate an apple.   ] (AI生成 / 教科書から選択)
品詞:     [名詞 ▾]
─────────────────────
[登録する]
```

- 表示要素: 入力フォーム（OCRタップ起点の場合は`text`に事前入力、`sourcePhoto`を保持）
- 主要アクション: 登録（同一`text`の既存`Word`がある場合は新規作成せず`WordOccurrence`のみ追加）
- 遷移先: 登録後 → 呼び出し元（2.1 ホーム／2.5 写真詳細／2.6 単語帳一覧）

### 2.9 復習（フラッシュカード）

```
3 / 12
┌─────────────────────┐
│                       │
│        apple          │
│                       │
│     (タップで裏面)     │
└─────────────────────┘
   [覚えてない]  [覚えた]
```

- 表示要素: `dueDate`が今日以前の`Word`を順に1件ずつ。表面`text`→タップで裏面
  `translation`/`exampleSentence`
- 主要アクション: 「覚えた」「覚えてない」で`reviewState`更新（`lastReviewedAt`/`reviewCount`/
  次回`dueDate`。アルゴリズム詳細は data-model.md 9章の通り未確定）
- 遷移先: 全件終了 → 2.6 単語帳一覧（完了サマリ表示）

### 2.10 問題一覧

```
[全体 ▾] [レッスン: Unit 3 ▾]
正答率 80% (5問中)
┌─────────────────────┐
│ 多肢選択  未回答        │→
│ 穴埋め    正解          │→
└─────────────────────┘
```

- 表示要素: `Question`一覧（`type`/未回答・正誤バッジ）、表示モード切り替え（全体／レッスン
  絞り込み）、正答率サマリ（2.13と同様の集計をミニ表示）
- 主要アクション: 問題タップで出題開始
- 遷移先: タップ → 2.11 出題、サマリタップ → 2.13 成績・演習履歴

### 2.11 出題（クイズ）

```
Q3. Choose the correct word.

I ___ an apple yesterday.

( ) eat   ( ) ate   ( ) eating

[回答する]
```

- 表示要素: `prompt`、`type`に応じたUI（multipleChoice: `choices`から選択／fillInBlank:
  テキスト入力／rearrange: 語句並べ替えUI）
- 主要アクション: 回答 → `QuizResult`作成（`userAnswer`/`isCorrect`を`correctAnswer`と照合）
- 遷移先: 回答後 → 2.12 採点結果・解説

### 2.12 採点結果・解説

```
✓ 正解！

解説: "ate" is the past tense of "eat"...

[次の問題へ]  [一覧に戻る]
```

- 表示要素: `isCorrect`、`explanation`
- 主要アクション: 次の問題へ（同一レッスン内の未回答`Question`があれば連続出題）、一覧に戻る
- 遷移先: 次の問題 → 2.11（再）、戻る → 2.1 ホーム（出題を2.1から始めた場合）／2.10（2.10から始めた場合）

### 2.13 成績・演習履歴

```
[全体 ▾] [レッスン: Unit 3 ▾]
正答率: 80% (5問中4問正解)

履歴
・10/14 多肢選択 ✓
・10/14 穴埋め   ✗
```

- 表示要素: `QuizResult`集計（正答率）と`answeredAt`順の履歴一覧、表示モード切り替え
  （全体／レッスン絞り込み）
- 主要アクション: 履歴タップで該当`Question`の採点結果（2.12相当）を再表示
- 遷移先: なし（参照のみ。再挑戦は2.10から）

### 2.14 設定

```
母語（翻訳先言語）: [日本語 ▾]
```

- 表示要素: 母語設定（仕様書6章の翻訳言語ペアに使用。対応言語リストは仕様書9章の未確定事項）
- 主要アクション: 母語の変更
- 遷移先: なし

## 3. 主要フローの確認（セルフレビュー）

- **毎日の主要フロー**: 2.1 ホーム起点で
  - (1) Lesson追加: `[+ 新しいレッスン]`（1タップ＋title入力）
  - (2) 写真アップロード: `[📷 写真を追加]` → 2.3 撮影 → 2.5 写真詳細（1タップ）
  - (3) 単語追加: `[📝 単語を追加]` → 2.5（OCRタップ）または2.8（1タップ）
  - (4) 問題を解く: `[✏️ 問題を解く]` → 2.11 出題（1タップ、未生成時は生成を挟む）
  - いずれもホームから1〜2タップで開始でき、クラス一覧・レッスン一覧のドリルダウンを
    経由しないことを確認
- **低頻度の管理操作**: クラス追加・既存レッスンへの切り替えは2.1ヘッダー → 2.2切り替え
  シート → `[+ クラスを追加]` / 各クラスの `[+]` で一通り行えることを確認
- 撮影 → OCR・翻訳: 2.3 撮影 → 2.5 写真詳細（OCR・翻訳結果を並べて表示）— 仕様書3.1章を満たす
- 単語帳登録 → 復習: 2.1/2.5/2.6 から登録（2.8）→ 2.7 単語詳細で確認 → 2.9 で復習 — 仕様書3.2章を満たす
- 問題作成 → 演習: 2.10 問題一覧 → 2.11 出題 → 2.12 採点・解説 → 2.13 成績で履歴確認 — 仕様書3.3章を満たす
- 全エンティティ（Class/Lesson/Photo/Word/WordOccurrence/Question/QuizResult）がいずれかの
  画面に表示されることを1章の対応表および本章のフローで確認済み
