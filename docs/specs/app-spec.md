# ESL Learning Assistant 仕様書（v1）

策定経緯: [docs/plans/archive/app-spec.md](../plans/archive/app-spec.md)

## 1. 概要

アメリカで ESL（English as a Second Language）の授業を受ける学習者が、授業中・授業後に使う
iPhone アプリ。教科書を撮影して文字起こし・翻訳し、知らない単語を単語帳に登録し、
教科書内容から練習問題を自動生成する。

## 2. 想定ユーザー

- アメリカで ESL クラスを受講している学習者
- 日本語ネイティブに限らず、母語が異なる学習者も想定する
  - UI／翻訳先言語は学習者が設定画面で指定した母語を使う（日本語固定にしない）

## 3. コア機能

### 3.1 撮影 → 文字起こし・翻訳（OCR & Translation）

- 教科書ページなどをカメラで撮影する（撮影自体はオフラインでも可能）
- 撮影した画像を Claude API に送信し、文字起こし（OCR）と翻訳を行う（オンライン時のみ）
- 文字起こし結果と翻訳結果を並べて表示する

### 3.2 単語帳（Vocabulary）

- 単語の登録方法
  - OCR結果のテキストをタップして登録
  - 自由入力で手動登録
- 単語ごとに保持する情報
  - 翻訳
  - 例文（教科書からの抜粋、または AI 生成）
  - 発音（TTS による音声読み上げ）
  - 品詞・文法情報
- 復習機能（間隔反復の復習クイズ）を持つ
  - 間隔反復は固定ステップの Leitner 方式（3日→7日→14日→30日→90日。正解で次ステップ、
    不正解で最初のステップに戻す）。詳細は [data-model.md](data-model.md) §5 `WordReviewState`
  - `dueDate <= 今日` の単語を「今日の復習」として Words タブから出題する
  - 問題は28形式（テキスト・音声・イラスト出題 × 4択・タイプ入力回答、問題文はすべて英語）。
    **backend が AI 生成してサーバに保存**し（1単語×1形式につき複数バリエーション）、
    アプリは復習セッション開始時にまとめて取得して、その中からランダムに出題する。
    サーバに問題が無い単語（オフライン・生成前）は出題しない。
    形式一覧は [docs/plans/archive/word-memorization-quiz.md](../plans/archive/word-memorization-quiz.md) §3.3、
    サーバ生成・保存の設計は
    [docs/plans/archive/quiz-questions-server-storage.md](../plans/archive/quiz-questions-server-storage.md)
  - AI 生成問題（3.3章、レッスン紐づけ）とは別機能。復習クイズは `Question` レコードを作らず、
    結果は単語の復習状態（`reviewState`）の更新のみで記録する

### 3.3 問題作成（Quiz Generation）

- 撮影・登録した教科書内容を元に、Claude API で練習問題を自動生成する
- 出題形式: 選択肢（多肢選択）問題／穴埋め問題／記述・並べ替え問題
- 難易度調整 UI は持たず、教科書内容に応じた難易度を AI に一任する
- 演習結果（正答率・履歴）を記録する

## 4. データの管理単位・保存方針

- 「クラス」と、その下の「レッスン（授業）」という 2 段階の単位を持つ
  - クラス: 受講しているコース・科目単位（例:「ESL Beginner A」）。複数レッスンをまとめる親単位
  - レッスン: クラス内の1回分の授業。日付等による自動区切りはせず、作成・切り替えはユーザーが手動で行う
- 写真・単語・問題は、以下 2 つの表示モードで参照できる
  - レッスンごとのビュー
  - 全体の一覧ビュー
- ユーザーデータ（クラス・レッスン・写真・単語・問題・成績履歴）は**端末内ローカル保存のみ**
  - クラウド同期・バックアップは対象外（将来検討事項）

## 5. アーキテクチャ

### 5.1 iOS アプリ

- ネイティブ iOS（Swift / SwiftUI）
- ユーザーデータはローカル（端末内）に保存
- OCR・翻訳・問題生成は、すべてオンライン時にバックエンド経由で Claude API を呼び出す
  （オンデバイス OCR は採用しない）

### 5.2 バックエンド

- Claude API へのリクエストを中継するバックエンドを用意する（API キーをアプリに直接持たせない）
- まずはローカル開発環境で動作させる（クラウドデプロイは将来検討）
- 通信ログを記録し、管理画面（Web）で閲覧できるようにする
  - 表示内容: トークン数・コスト、画像と OCR 結果の対応、単語と翻訳の対応、生成された問題

## 6. 言語設定

- 翻訳の言語ペア: 英語 ⇔ 学習者が設定画面で指定した母語
- 対応言語の範囲は MVP では未確定（実装フェーズで決定）

## 7. 開発フェーズ（優先順位）

1. **Phase 1**: 撮影 → OCR・翻訳
2. **Phase 2**: 単語帳（登録・翻訳・復習）
3. **Phase 3**: 問題作成

各 Phase ともに、対応するバックエンド機能・管理画面（通信ログ表示）を含めて実装する。

## 8. 対象外（Non-goals / 将来検討）

- クラウドへのユーザーデータ同期・複数端末利用
- バックエンドの本番クラウドデプロイ（まずはローカルのみ）
- 問題の難易度を手動調整する UI
- オフラインでの OCR・翻訳・問題生成（ネットワーク必須）
