# 管理画面デザイン刷新

## 目的・背景

管理画面（`/admin` 以下）は現在、素の `sans-serif` + 罫線テーブルのみの最小スタイル（`backend/src/admin.ts` の `PAGE_STYLE`）。
機能は揃ってきたので、見た目をカッコ良くしたい。いきなり実装せず、複数のデザイン案をモックアップで作成・比較検証してから採用案を決める。

## 現状の構成

- 管理画面は `backend/src/admin.ts` 一枚で、Express ルーターがインライン HTML 文字列を返すサーバレンダリング構成
- 共通スタイルは `PAGE_STYLE` 定数、ページ固有スタイルは `renderPage(title, extraStyle, body)` の第2引数
- ページ構成:
  - OCR・翻訳ログ 一覧 (`/admin`) / 詳細 (`/admin/logs/:id`)
  - 単語情報ログ 一覧 (`/admin/word-info`) / 詳細 (`/admin/word-info/:id`)
  - 単語一覧 (`/admin/words`) / 詳細 (`/admin/words/:id`、再生成・削除ボタンあり)
  - TTS一覧 (`/admin/tts`、audio プレイヤー・削除ボタンあり)
  - システムログ (`/admin/system-logs`)
- 静的アセットは無し。CSS はすべてインライン `<style>`

## 対応方針

### Phase 1: デザイン例の作成・検証（このタスクの範囲）

代表ページ（OCR・翻訳ログ一覧）を題材に、方向性の異なる 3 案のモックアップを作成し、1 ページで切り替え比較できる形にまとめる。

- 案A: **クリーンライト** — 白ベースのカード + 控えめなシャドウ、SaaS 管理画面風（Linear / Stripe 系）
- 案B: **ダークオプス** — ダーク背景 + モノスペースアクセント、ログビューア・ターミナル風
- 案C: **サイドバーダッシュボード** — 左固定サイドバーナビ + 上部サマリーカード、王道ダッシュボード風

各案とも実データに近いサンプル（日時・モデル名・コスト・ステータス等）を入れ、ナビ・テーブル・ステータス表示・詳細リンクの見え方まで確認できるようにする。作成後、各案の長所短所を評価し推奨案を提示する。

### Phase 2: 採用案の実装（完了: 2026-07-03）

ユーザー選定の結果、**案Bのダーク配色 × 案Cのサイドバーレイアウト**のハイブリッドを採用。
`admin.ts` の `PAGE_STYLE` をダークテーマに全面差し替え、`navLinks()` を `sidebar()`（左固定ナビ）に置換、
`renderPage()` にサイドバー付きレイアウトと viewport meta を追加。一覧ページにはサマリーカードを実装
（OCR: 件数/コスト/エラー/平均処理時間、単語情報: +キャッシュ返却、TTS: 件数/料金/合計サイズ）。
全ページを実データ + headless Chrome で目視確認済み。

### （参考・当初案）Phase 2: 採用案の実装（別タスク・ユーザー選定後）

- 採用案の CSS を `PAGE_STYLE` に反映し、`renderPage` / `navLinks` を共通レイアウト（ヘッダー or サイドバー）に対応させる
- 一覧 5 ページ + 詳細 3 ページすべてに適用
- 機能・ルーティングは変更しない（見た目のみ）

## 影響範囲

- Phase 1: コード変更なし（モックアップのみ）
- Phase 2: `backend/src/admin.ts` のスタイル・レイアウト部分のみ。API・DB は触らない

## テスト方針

- Phase 1: モックアップを目視確認
