# 同じクラスに同名のレッスンを作れないようにする

## 目的・背景

現状 `LessonAddView` は入力されたレッスン名をそのまま登録するため、同じクラス内に
同名のレッスンを複数作れてしまう。切り替えシートで区別できず誤選択の元になるので、
同一クラス内でのレッスン名の重複を禁止する。

## 対応方針

- レッスン作成は `LessonAddView` の1箇所のみ（`grep "Lesson(schoolClass"` で確認済み）
- `LessonAddView` に重複チェックを追加する
  - 対象クラス（`schoolClass.lessons`）内に、トリム後のタイトルと大文字小文字を区別せず
    一致するレッスンがあれば重複とみなす（単語の重複判定 `WordAddView` と同じ方式）
  - 重複時は「追加」ボタンを無効化し、フォームのフッターに赤字で理由を表示する
  - 念のため `addLesson()` 側でも重複時は早期 return する（多重防御）

## 影響範囲

- `ios/ESLLearningAssistant/Sources/Views/LessonAddView.swift` のみ

## テスト方針

- UI テストに「既存と同名のレッスン名を入力すると追加ボタンが無効になり、
  重複メッセージが表示される」ケースを追加する
- 既存 UI テスト（レッスン作成を含むフロー）が壊れていないことを確認する
