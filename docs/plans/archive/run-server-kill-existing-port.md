# run-server.shに既存ポート占有プロセスの停止機能を追加

## 目的・背景

直前のセッションで、Claude側がバックグラウンドでバックエンドを起動したままの状態でユーザーが
`./run-server.sh` を実行し `EADDRINUSE: address already in use :::8801` で失敗する事象が発生した。
`run-server.sh` 実行時に対象ポートを既に使っているプロセスがあれば自動的に停止してから
起動し直せるようにする。

## 対応方針

- `backend/.env` の `PORT`（未設定なら `config.ts` と同じ既定値 8801）を読み取る。
- `lsof -tiTCP:<port> -sTCP:LISTEN` で当該ポートをLISTENしているPIDを検出する。
- 見つかった場合は `kill` (SIGTERM) を送り、最大数秒待ってポートが解放されるのを確認する。
  解放されなければ `kill -9` で強制終了する。
- 上記処理は `.env` チェックの直後、`npm install`/`build`/`start` の前に実施する。

## 影響範囲

- `run-server.sh`

## テスト方針

- 既にポートを使用しているダミープロセスを起動した状態で `run-server.sh` を実行し、
  自動的に停止されてサーバーが正常起動することを確認する。
- ポート未使用の状態でも問題なく起動することを確認する。
