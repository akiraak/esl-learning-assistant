# 管理画面: 音声文字起こしログ一覧にタイトル列を追加

## 目的・背景

コンテンツファイル一覧（audio タブ）にはアプリ側タイトル（AudioClip.title）を
表示しているが、音声文字起こしログ（/admin/transcriptions）の一覧には出ていない。
どの音声のログか分かりやすいよう、文字起こしログ一覧にもタイトルを表示する。

`transcription_requests.title` に記録済みなので表示のみの変更で済む
（コンテンツファイル一覧と違いファイル名での突き合わせも不要で、行自体が title を持つ）。

## 対応方針

- `/admin/transcriptions` の一覧テーブルに「タイトル」列を追加（日時の次）
- 表示はコンテンツファイル一覧と同じ作法: title があれば表示、無ければ faint の「—」
  （旧アプリからの送信や title 追加前のレコードは NULL のため）

## 影響範囲

- backend の `src/admin.ts`（一覧テーブルの1列追加）のみ。iOS・API・DB 変更なし

## テスト方針

- HTML テンプレートのみの変更のため単体テストは追加しない
- E2E: サーバ再起動し、title ありレコード（#4）でタイトル表示、
  title なしレコード（#3）で「—」表示を curl / スクリーンショットで確認
