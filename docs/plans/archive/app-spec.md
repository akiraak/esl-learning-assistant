# アプリ仕様策定プラン

## 目的・背景

README には ESL 学習全般（授業準備・授業の振り返り・語彙練習・ライティング支援・スピーキング練習）を
支援する AI アプリと記載されているが、ユーザーからは「アメリカで ESL の授業を受ける際にサポート的に使う
iPhone アプリ」という、より具体的な利用シーン（授業中のリアルタイム支援）が提示された。
現時点で仕様は未確定のため、ユーザーとの対話を通じて要件を詰める。

## 対応方針

- AI（Claude）との会話形式で要件をヒアリングし、以下を段階的に確定する
  - 想定ユーザー像・利用シーン（授業中／授業前後）
  - コア機能（リアルタイム字幕、辞書・翻訳、メモ支援、発話サポート等）
  - 動作環境（オフライン要否、ネットワーク・API 利用方針）
  - プラットフォーム実装方式（ネイティブ iOS app / Web app 等）
  - UI/UX の言語（日本語ベースか多言語対応か）
- 決定事項はこのファイルおよび `docs/specs/` に随時反映する
- 仕様確定後、`docs/specs/app-spec.md`（仮）として正式な仕様書を作成する

## 影響範囲

現時点でアプリのコードは存在しない（リポジトリは vibeboard 等の開発管理ツールのみ）。
本タスクはコード変更を伴わない要件定義フェーズ。

仕様策定フェーズのため、テスト対象なし。仕様確定後の実装フェーズでテスト方針を別途定義する。

## 決定事項（ヒアリング結果）

### コア機能（授業中に支援したい内容）

1. **教科書の写真 → 文字起こし＋翻訳**: 教科書ページを撮影し、OCR でテキスト化、翻訳を表示する
2. **単語の登録と翻訳**: 知らない単語を登録し、翻訳を保存できる（個人の単語帳的機能）
3. **教科書を元にした問題作成**: 撮影・登録した教科書内容から練習問題を自動生成する

### 想定利用者

- 日本語ネイティブ以外の ESL 学習者も想定する → UI・翻訳先言語は日本語固定にせず、多言語を見据えた設計にする
  （学習者の母語に応じて翻訳先言語を切り替えられる必要がある）

### 通信環境・オフライン要否

- **オフラインで動作させたい**: 写真撮影、文字入力
- **オンライン必須**: 翻訳、文字起こし（OCR/AI 処理）、問題生成などのAI機能
- → 撮影・入力はローカルで即時実行し、AI処理（OCR・翻訳・問題生成）はオンライン時にAPIへ送信する設計。
  オフライン時はキューイングして後でオンラインになったら処理する案も検討

### 単語帳機能の詳細

翻訳に加え、以下をすべて持たせる:
- 例文（教科書からの抜粋 or AI生成）
- 発音（TTS による音声読み上げ）
- 復習機能（フラッシュカード／間隔反復）
- 品詞・文法情報

### 問題作成機能の詳細

出題形式は以下をすべて想定:
- 選択肢（多肢選択）問題
- 穴埋め問題
- 記述・並べ替え問題

### データ管理単位

表示モードを複数持たせる:
- 授業（レッスン）ごとのビュー
- 全体の一覧ビュー
（同じデータを異なる切り口で見られるようにする。授業という概念自体は持つ）

### データ保存先

ローカル保存のみ（端末内）。クラウド同期・バックアップは対象外。

### 技術スタック

- **プラットフォーム**: ネイティブ iOS（Swift/SwiftUI）
- **OCR方式**: オンライン・AI処理に含める（オンデバイスOCRは採用しない。撮影自体はオフライン可だが、文字起こしは翻訳・問題作成と同じくオンライン時にAI処理する）
- **AIバックエンド**: Claude API（Anthropic）— OCR・翻訳・問題作成を一貫して担う
- **レッスンの区切り**: ユーザーが手動で作成・選択する（自動区切りはしない）

### バックエンド・API呼び出し方式

- アプリ単体でAPIキーを直接持つのではなく、**バックエンドを用意**する
- バックエンドは Claude API へのリクエストを中継し、**通信ログを管理画面で閲覧できる**ようにする
  （APIキーの秘匿、リクエスト/レスポンスの可視化・デバッグ用途）
- → 「データ保存はローカルのみ」はユーザーデータ（レッスン・写真・単語・問題）の話であり、
  AI呼び出し用バックエンド／通信ログは別の関心事として併存する

### 翻訳の言語ペア

- 英語 ⇔ 学習者が指定した母語（設定画面で選択）

### MVP開発方針

- 段階的に作る。3つのコア機能（撮影OCR・翻訳／単語帳／問題作成）に優先順位をつけて順次実装する
  （優先順位は次回ヒアリングで確定）

### 開発優先順位（段階的MVP）

1. 撮影OCR・翻訳（最初に作る）
2. 単語帳（登録・翻訳・復習）
3. 問題作成

### バックエンド稼働環境

まずはローカル開発環境で動かす（クラウドデプロイは将来検討）。

### 問題作成の難易度

難易度調整UIは不要。教科書内容に応じた難易度にAIが自動で任せる。

### 単語登録フロー

- OCR結果の文字をタップして登録
- 自由入力での追加も可能（両方対応）

### 問題演習の成績記録

正答率・履歴を記録する。

### 通信ログ管理画面の表示内容

- トークン数・コスト
- 画像とOCR結果の対応
- 単語と翻訳の対応
- 生成された問題
