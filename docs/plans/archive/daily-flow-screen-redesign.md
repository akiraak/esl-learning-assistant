# 日常フロー中心の画面構成見直しプラン

## 目的・背景

[docs/specs/screen-design.md](../specs/screen-design.md)（v1）はデータモデルの全エンティティを
網羅する形で画面を洗い出したが、「クラス一覧→レッスン一覧→レッスンホーム」のドリルダウンが
毎回必要で、日常的な操作には遠回りだった。

ユーザーからのフィードバック: 毎日の主な使い方は次の4ステップ。
1. Lesson追加
2. 写真をアップロード
3. 単語追加
4. 問題を解く

これを最短導線にしつつ、クラス追加などの低頻度な管理操作にも対応できる構成に見直す。

## 対応方針

- ボトムタブの「レッスン」タブを「ホーム」に変更し、ドリルダウンせず常に「現在のレッスン」を
  表示するダッシュボードにする（現在のレッスン＝直近に作成・選択した `Lesson`。端末内のUI状態
  として保持し、データモデルへのフィールド追加は行わない）
- 旧「クラス一覧」「レッスン一覧」画面は、ホームのヘッダーから開く1枚のボトムシート
  （クラス/レッスン切り替え）に統合し、独立画面・ドリルダウン階層をなくす
- ホームに4ステップに対応するクイックアクションを常設する
  - 新しいレッスン作成（ヘッダー直下、`title`入力のみ）
  - 写真を追加（撮影画面へ直行）
  - 単語を追加（直近写真のOCRタップ or 手動登録へ直行）
  - 問題を解く（未回答があれば出題へ直行、未生成なら生成してから出題）
- 単語帳・問題・設定タブ、各詳細画面（写真詳細/単語詳細/出題/採点/成績等）の役割は変更しない。
  画面番号の振り直しと、ホームからの遷移元表記の更新のみ行う
- 成果物は既存の `docs/specs/screen-design.md` を改訂する（新規ファイルは作らない）

## 影響範囲

- 変更: `docs/specs/screen-design.md`（0章ナビゲーション、1章対応表、2章の画面構成・番号、
  3章フロー確認を改訂）
- 変更: `TODO.md` / `DONE.md`（タスク移動）
- コード変更なし（実装はまだ着手していないため）

## テスト方針

- ドキュメントタスクのためテストコードはなし
- 「毎日の4ステップ」がホーム画面から最短1〜2タップで開始できる導線になっているかをセルフレビュー
- クラス追加・レッスン切り替えなど低頻度操作が引き続き行えることを確認
- 全エンティティ（Class/Lesson/Photo/Word/WordOccurrence/Question/QuizResult）がいずれかの画面
  から参照されていることを再確認
