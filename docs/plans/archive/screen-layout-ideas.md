# 画面レイアウトのアイデア出しプラン

## 目的・背景

[docs/specs/app-spec.md](../specs/app-spec.md) 9章「未確定事項」に挙げた
「画面構成の詳細（ワイヤーフレーム）」を検討する。
[docs/specs/data-model.md](../specs/data-model.md) でデータモデル（Class / Lesson / Photo /
Word / WordOccurrence / Question / QuizResult）が確定したため、これを元に必要な画面を洗い出し、
各画面でどのデータをどう見せるかのレイアウト案を出す。
本タスクはアイデア出し・ドキュメント作成のみで、SwiftUI 実装は対象外（実装は各 Phase の実装タスクで行う）。

## 対応方針

- データモデルの各エンティティ・関連と、仕様書3章のコア機能（撮影/単語帳/問題作成）・
  4章の表示モード（レッスンごとのビュー／全体の一覧ビュー）を突き合わせて画面を洗い出す
- Phase 1〜3（仕様書7章の開発フェーズ）に沿って画面をグルーピングする
  - Phase 1: クラス/レッスン管理、撮影 → OCR・翻訳
  - Phase 2: 単語帳（登録・一覧・詳細・復習）
  - Phase 3: 問題作成・演習・結果
- 各画面について、表示する主要情報・参照するエンティティ・主要アクション・画面遷移先を
  テキストベースのワイヤーフレーム（箇条書き／簡易レイアウト）として記述する
- 細部のビジュアルデザイン（配色・フォント等）は対象外。情報設計・画面構成・遷移に絞る
- 成果物は `docs/specs/screen-design.md` に新規作成し、`app-spec.md` 9章から参照を追加する

## 影響範囲

- 新規: `docs/specs/screen-design.md`
- 変更: `docs/specs/app-spec.md`（9章に参照リンクを追記）
- 変更: `TODO.md` / `DONE.md`（タスク移動）
- コード変更なし

## テスト方針

- ドキュメントタスクのためテストコードはなし
- データモデルの全エンティティ（Class / Lesson / Photo / Word / WordOccurrence / Question /
  QuizResult）が、いずれかの画面から参照・表示されているかをセルフレビューする
- 仕様書3章のコア機能（撮影→OCR・翻訳、単語帳、問題作成）の主要フローが画面遷移として
  一通りたどれるかを確認する

## Phase / Step

- Phase 1: 画面一覧・データ対応表の作成
  - データモデルの各エンティティ／仕様書3〜4章の機能を画面候補にマッピングする
- Phase 2: 各画面のレイアウト案作成
  - Phase 1 でリストアップした画面ごとに、表示要素・主要アクション・遷移先を記述する
- Phase 3: 仕様書への反映
  - `docs/specs/screen-design.md` として清書し、`app-spec.md` 9章にリンクを追加する
