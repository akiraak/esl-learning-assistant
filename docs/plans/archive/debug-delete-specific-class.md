# デバッグメニュー: クラス削除をクラス指定式に変更

## 目的・背景

デバッグメニューの「クラスとそのレッスンの全削除」は全クラスを一括削除する仕様だが、
特定のクラスだけ消したいケースに対応できない。削除するクラスを指定できるようにする。

## 対応方針

- ボタン名を「クラスとそのレッスンの削除」に変更し、タップで確認ダイアログに
  クラス一覧（クラス名＋レッスン数）を destructive ボタンとして列挙する
  - クラスのボタンをタップすると、そのクラスだけ削除する（ダイアログ自体が確認を兼ねる。
    メッセージに「元に戻せない」旨を明記）
  - ダイアログ末尾に「すべてのクラスを削除」も残し、従来の一括削除もできるようにする
  - クラスが 0 件のときはボタンを無効化する
- `DebugDataCleaner.deleteClass(_:context:)` を追加する
  - 削除前に対象クラス配下の写真ファイル名を集め、エンティティ削除・保存後に
    `PhotoStorage.delete(fileName:)` で該当ファイルだけ削除する
    （全削除と違いディレクトリごと消せないため）

## 影響範囲

- `ios/ESLLearningAssistant/Sources/Support/DebugDataCleaner.swift`（`deleteClass` 追加）
- `ios/ESLLearningAssistant/Sources/Views/SettingsView.swift`（ダイアログをクラス選択式に変更）

## テスト方針

- ユニットテスト: クラス 2 件（各レッスン・写真・単語出現あり）を用意し、片方を
  `deleteClass` で削除 → 削除した側の Lesson / Photo / WordOccurrence だけ消え、
  もう片方と Word は残ることを検証する
- UI テスト: 全クリア → クラス作成 → 「クラスとそのレッスンの削除」でそのクラスを指定して
  削除 → レッスンタブが空状態に戻ることを検証する
- 既存のユニットテスト・UI テスト（全クリア）が壊れていないことを確認する
