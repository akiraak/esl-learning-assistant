# 復習クイズ回答後の正解ハイライトを目立たせる（不正解も調整）

## 目的・背景

- TODO: 「単語回答後の正解の緑が見ずらい。もっと目立つように」＋不正解の表示も目立たなければ色を調整する
- 現状（`ReviewSessionView`）:
  - 選択肢（テキスト4択 `choiceButtons` / イラスト4択 `illustrationChoiceGrid`）は回答後、
    正解に `.green.opacity(0.25)`、選んだ誤答に `.red.opacity(0.25)` の背景を塗るのみ
    （`choiceBackground`、349-354行）
  - テキスト4択の枠線は正誤にかかわらず `.quaternary` のままで強調されない（301-304行）
  - イラスト4択のみ正解に緑3ptの枠線が付くが、選んだ誤答の枠線はセパレータ色のまま
- 淡い25%の色ではどれが正解だったか一目で分かりにくい、という報告

## 対応方針

`ReviewSessionView` のみの見た目調整（ロジック・レイアウト構造は変えない）。

1. **背景の強調**: `choiceBackground` の不透明度を 0.25 → 0.4 に引き上げ（正解・誤答とも）
2. **枠線の強調（テキスト4択）**: 回答後は正解に `Color.green`、選んだ誤答に `Color.red` の
   2.5pt 枠線を表示（それ以外は従来の `.quaternary` 1pt）
3. **アイコン追加（テキスト4択）**: 正解の行末に `checkmark.circle.fill`（緑）、選んだ誤答の行末に
   `xmark.circle.fill`（赤）を表示。色弱でも形で判別できるようにする
4. **イラスト4択の対称化**: 正解の緑3pt枠線は維持し、選んだ誤答にも赤3pt枠線を追加。
   さらに右上に正誤アイコンバッジ（checkmark/xmark）を重ねる
5. フィードバックカード（Correct!/Incorrect の文字色）は現状で十分目立つため変更しない

## 影響範囲

- 変更: `ios/ESLLearningAssistant/Sources/Views/ReviewSessionView.swift` のみ
- 出題ロジック・習熟度計算・アクセシビリティ識別子は不変

## テスト方針

- `xcodebuild build` でコンパイル確認
- シミュレータで実セッションを実施して見た目を確認:
  - シミュレータ既存単語をローカル backend に問題ストックのある単語へ一時リネームし、
    接続先をローカル backend にして復習セッションを起動（検証後に原状復帰）
  - 正解時・不正解時の選択肢ハイライト（背景・枠線・アイコン）をスクリーンショットで確認
