# 管理画面: 一覧＋詳細ページ方式へのUI改善

## 目的・背景

現状の `/admin` は1テーブルにID・日時・画像サムネ・OCR結果・翻訳結果・モデル/トークン・コスト・状態・処理時間を
すべて詰め込んでおり、OCR結果・翻訳結果のセルは320px×240pxでスクロールする作りになっている。
レッスン画像のOCR結果を精度確認も含めてちゃんと読めるようにしたいというのが今回の必須要件。

ユーザーとレイアウト方針を相談した結果、「一覧＋詳細ページ方式」を採用する。
- 一覧（`/admin`）: ID・日時・縮小画像・状態・詳細へのリンクのみのコンパクトな表
- 詳細ページ（`/admin/logs/:id`）: 大きめの画像とOCR結果全文・翻訳結果全文を余白たっぷりで表示

## 対応方針

### Step 1: 一覧ページをコンパクトな表に変更する

- `backend/src/admin.ts` の `GET /admin` から OCR結果・翻訳結果カラムを削除し、
  ID・日時・縮小画像・モデル/トークン・コスト・状態・処理時間・詳細リンクのみの表にする。
- 各行に `/admin/logs/:id` への詳細リンクを追加する（行全体をクリック可能にする）。

### Step 2: 詳細ページを新設する

- `GET /admin/logs/:id` ルートを追加し、`getRequestLog(id)` で1件取得して表示する。
- 画像は120x120サムネではなく、大きめ（例: max-width 480px程度）で表示し、
  クリックで原寸画像を新規タブで開けるようにする（`<a href="/admin/logs/:id/image" target="_blank">`）。
- OCR結果・翻訳結果は幅・高さ制限（overflow scroll）を外し、ページ全体の横幅を使って
  折り返し表示する。フォントサイズも一覧より大きめにする。
- モデル/トークン数・コスト・状態・エラーメッセージ・処理時間などのメタ情報も表示する。
- 一覧に戻るリンクと、前後のログ（id+1が新しい・id-1が古い）への簡易ナビゲーションを付ける
  （`getRequestLog` で存在チェックしてリンクの有無を切り替える）。

### Step 3: 共通HTML head/styleの重複を整理する

- 一覧・詳細で共通する `<head>` のスタイル定義をヘルパー関数化し、重複を避ける。
- Markdownレンダリング（`renderMarkdown`）・XSSエスケープ処理は既存のものをそのまま流用する。

## 影響範囲

- `backend/src/admin.ts`

## テスト方針

- `npm run build`（backend）でビルド確認。
- `run-server.sh` でサーバー再起動後、`curl http://localhost:8801/admin` で一覧がコンパクトに
  表示されること、`curl http://localhost:8801/admin/logs/<id>` で詳細ページにOCR全文・翻訳全文が
  折り返し表示されることを確認する。
- 存在しないIDでの詳細アクセス（404表示）も確認する。
